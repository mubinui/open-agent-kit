# Orchestration Service API Endpoints
# Base URL: http://localhost:8000

================================================================================
ROOT
================================================================================
GET  /                                    - Root endpoint with API information
GET  /docs                                - Swagger UI documentation
GET  /openapi.json                        - OpenAPI specification

================================================================================
HEALTH & METRICS (/api/v1)
================================================================================
GET  /api/v1/health                       - Health check
GET  /api/v1/health/ready                 - Readiness check
GET  /api/v1/health/live                  - Liveness check
GET  /api/v1/metrics                      - Prometheus metrics
GET  /api/v1/metrics/dashboard            - Dashboard metrics

================================================================================
AUTHENTICATION (/api/v1/auth)
================================================================================
GET  /api/v1/auth/config                  - Get public authentication configuration
POST /api/v1/auth/token                   - Login for access token
POST /api/v1/auth/users                   - Create user
GET  /api/v1/auth/users/me                - Get current user information
POST /api/v1/auth/api-keys                - Create API key
GET  /api/v1/auth/api-keys                - List API keys
DELETE /api/v1/auth/api-keys/{api_key_id} - Revoke API key
POST /api/v1/auth/api-keys/{api_key_id}/rotate - Rotate API key

================================================================================
SESSIONS - CHATBOT ENDPOINTS (/api/v1/sessions)
================================================================================
POST /api/v1/sessions                     - Create new chat session
GET  /api/v1/sessions                     - List all sessions (optional: ?user_id=xxx&active_only=true)
GET  /api/v1/sessions/chats               - Get all chats for user (simplified DTO: id, title, messageCount, createdAt, updatedAt)
GET  /api/v1/sessions/user/{user_id}      - Get all sessions for a specific user (detailed)
GET  /api/v1/sessions/{session_id}        - Get session details
DELETE /api/v1/sessions/{session_id}      - Delete session
GET  /api/v1/sessions/{session_id}/history - Get chat history
POST /api/v1/sessions/{session_id}/messages - Send message to chatbot (main chat endpoint)

================================================================================
CHATBOT USAGE EXAMPLE
================================================================================

1. Create a session:
   POST /api/v1/sessions
   Body: { "workflow_id": "procurement_chatbot", "user_id": "john.doe" }
   Response: { "session_id": "uuid-here", ... }

2. Send messages:
   POST /api/v1/sessions/{session_id}/messages
   Body: { "message": "What is the status of requisition REQ-2024-001?" }
   Response: { "response": "...", "chat_history": [...] }

3. Get chat history:
   GET /api/v1/sessions/{session_id}/history

4. Get all chats for a user (simplified DTO for frontend):
   GET /api/v1/sessions/chats?user_id=john.doe
   Response: {
     "chats": [
       {
         "id": "uuid-here",
         "title": "What is the status of requisition...",
         "messageCount": 5,
         "createdAt": "2024-12-10T08:00:00Z",
         "updatedAt": "2024-12-10T08:30:00Z"
       }
     ],
     "total": 1
   }

5. Get all sessions for a user (detailed):
   GET /api/v1/sessions/user/john.doe
   Response: { "user_id": "john.doe", "sessions": [...], "total_count": 5, "active_count": 2 }

6. List all sessions (with optional filters):
   GET /api/v1/sessions?user_id=john.doe&active_only=true
   Response: [{ "session_id": "...", "workflow_id": "...", ... }]

7. End session:
   DELETE /api/v1/sessions/{session_id}
